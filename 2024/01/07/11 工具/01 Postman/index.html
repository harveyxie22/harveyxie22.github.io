Postman

Harvey Xie 2024-01-07 10:48:00
Categories： 工具 Tags： Postman

介绍

什么是 Postman

Postman 是一款用于测试和开发 API 的工具。它提供了一个直观的界面，让开发人员能够轻松地发送 HTTP 请求、查看响应结果以及进行调试和验证。通过 Postman，开发人员可以模拟不同类型的请求（如 GET、POST、PUT、DELETE 等），设置请求头和参数，并查看服务器返回的响应数据。它还支持创建和管理集合、环境变量、测试脚本等功能，使得 API 测试和开发过程更加高效和方便。无论是单个开发者还是团队协作，Postman 都是一款非常实用的工具。

发展历史

Postman 的两个联合创始人 Abhinav Asthana 和 Ankit Sobti 最开始是在 Yahoo 构建一个应用程序的前端架构，那时他们必须使用 API。他们意识到缺少支持 API 开发的工具，于是坐下来编写代码解决这个问题，并将其发布在谷歌的网上商店——这就是 Postman 最早的由来。

2012 年，在谷歌的网络商店上，Postman 已经有了将近 2000 次下载。于是他们开始更加仔细地研究这个领域，发现人们迫切的需要一个 API 工作流，从概念化到文档的发布再到 API 的传播。

于是， Postman 就从一个业余项目发展成为了一个许多世界顶级组织使用的 API 平台。截止到目前现在全世界有已经有超过 600 万的开发者在使用 Postman。

为什么要使用 Postman

首先在项目初期，测试人员提前介入，进行接口测试，模拟客户端与服务端的交互。有问题提前抛出来。保证接口调用是没问题的。其次，接口测试完成之后再进行系统测试，会轻松很多。明白了各个接口在做什么，各个参数的模拟实际上就是业务场景的模拟。系统测试出现一些问题可以更快速的定位是客户端还是服务端出问题。

另外，接口是获取和操作资源的方式，而大部分系统和产品中，资源一般都是产品的核心，比如微信核心资源就是通讯录关系链和聊天记录等，因此资源是必测的。

而接口中大部分的内容是数据，通过数据的对比我们能推测到系统和产品的逻辑，测接口就是测逻辑。最后接口中的返回相对单纯，不像 web 页面，HTML 代码中有太多 UI 的东西，UI 最不稳定，变化太快，接口相对稳定一点点，但是里面的干扰信息更少，断言相对容易很多。

下载和安装

下载

  1. 访问 Postman 官方网站：https://www.postman.com/
  2. 在网站首页，点击 “Download the App” 按钮。
  3. 在下载页面，你可以选择适用于不同操作系统的版本，比如 Windows、Mac 或 Linux。
  4. 点击对应操作系统的下载按钮，然后保存下载文件到你的计算机上。

安装

  1. 下载完成后，找到下载的安装程序文件，并双击运行。
  2. 按照安装程序的指示，选择安装位置和其他设置。
  3. 安装完成后，你可以在开始菜单或启动器中找到 Postman 图标，双击打开应用程序。
  4. 首次打开 Postman，你需要创建一个免费的帐户。按照提示进行注册。
  5. 注册完成后，你可以登录到 Postman 应用程序中，开始使用它进行 API 测试和开发。

基本配置

语言和主题配置

在 Postman 的界面上方的右侧，可以找到 “Settings” 按钮，点击后弹出菜单，可以选择界面语言和主题。默认语言为英文，可以切换为其他语言，如中文、法语、德语等。主题包括亮色和暗色两种模式，可以根据个人喜好进行选择。

请求参数设置

当创建请求时，可以设置请求的 HTTP 方法（GET、POST、PUT、DELETE 等）、URL 地址和参数。在参数中，可以添加查询参数、请求头、表单数据等信息。如果需要添加更复杂的参数，也可以使用 JSON 或 XML 格式。

响应结果显示设置

在发送请求后，可以在 Postman 中查看服务器返回的响应结果。可以选择不同的显示方式，如原始文本、格式化 JSON、XML、HTML 等。如果响应结果很长，还可以将其折叠或缩短，以方便查看。

环境变量设置

Postman 支持环境变量的设置和使用。可以在 “Manage Environments” 菜单中创建和管理多个环境变量，如开发环境、测试环境、生产环境等。通过使用环境变量，可以轻松地在不同的环境中进行测试和开发。

接口集合管理

在 Postman 中，可以将多个接口归为一个集合，以方便管理和调用。可以通过创建不同的接口集合，对不同的项目或模块进行区分。同时，还可以在接口集合中添加测试和脚本等功能。

自动化测试设置

Postman 支持使用 JavaScript 编写测试脚本，可以自动化地对接口进行测试和验证。在测试脚本中，可以添加各种断言和条件判断，以检查接口是否符合预期。在 “Tests” 选项卡中，可以编写和执行测试脚本。

其他设置

Postman 还支持其他一些常用的设置，如请求代理、网络连接、SSL 证书等。可以根据实际需要进行配置和修改。

发送请求和查看响应

以下是使用 Postman 发送请求和查看响应的基本步骤：

  1. 打开 Postman 并创建请求： 打开 Postman，单击左上角的 “New” 按钮，选择请求方法（如 GET、POST、PUT 等），输入 URL 和其他请求参数（如请求头、查询参数、表单数据等）。
  2. 发送请求并查看响应： 单击请求下方的 “Send” 按钮，Postman 会发送该请求，并在右侧窗口中显示服务器返回的响应。响应结果包括状态码、响应头、响应体等信息。可以通过下拉框选择不同的显示方式，如原始文本、JSON 格式化、XML 格式化等。
  3. 查看请求历史记录： 在 Postman 的左侧导航栏中，单击 “History” 选项卡，可以查看所有发送过的请求的历史记录。可以选择某个请求并查看其详细信息，包括请求参数、响应结果、测试结果等。
  4. 使用收藏夹保存常用请求： 在 Postman 中，可以将常用的请求添加到收藏夹中，方便日后使用。在请求右侧的 “Save” 下拉箭头中，选择 “Add to collection”，然后选择或创建一个收藏夹即可。
  5. 使用环境变量进行请求参数设置： 在 Postman 中，可以使用环境变量来设置请求的参数，以方便在不同的环境中进行测试和开发。在请求参数中，可以使用环境变量名替代具体的值，如 /2024/01/07/11 工具/01 Postman/，其中 base_url 和 path 是环境变量。在设置环境变量时，需要在 “Manage Environments” 菜单中进行添加和管理。

请求参数

在 Postman 中请求参数的位置可以放到 Params 或者 Body 中。两者的区别在于请求参数在 HTTP 协议中的位置不一样。

Params

当使用 Params 进行参数请求时，参数会被置于 URL 中。

Body

当使用 Body 进行参数请求时，参数会被置于请求体中。在 Body 中可以设置不同的请求格式：none、form-data、x-www-form-urlencoded、raw、binary、GraphQL 等。

具体请求格式的特点如下表所示：

参数格式 特点 注意
multipart/form-data key-value 格式输入，通常文件传输使用该格式 在 Springboot 中不可以使用 @RequestBody 进行接收
application/x-www-form-urlencoded key-value 格式输入，但是不支持文件传输，与 multipart/form-data 的区别在于 HTTP 请求 body 的格式不一样。application/x-www-form-urlencoded 不可以设置 content-type，而 multipart/form-data 可以设置每个 value 的 content-type。 在 Springboot 中不可以使用 @RequestBody 进行接收
raw - Text：请求头是：text/plain
- JavaScript：请求头是：application/javascript
- JSON：请求头是：application/json
- HTML：请求头是：text/html
- XML：请求头是：application/xml		 如果希望传递的是 json 格式数据，选择 JSON 即可；在使用该方式请求，Springboot 必须有 @RequestBody 进行接收
binary 相当于 content-type: application/octet-stream。只可以上传二进制文件。 通常用此方式上传文件。但是由于没有键值对，所以一次只能上传一个文件

当使用 multipart/form-data 或者 application/x-www-form-urlencoded 进行传输时，在表单中直接填写 key-value，Springboot 会通过字段匹配的方式进行参数的解析。

@PostMappping("\test")
public String test(String name, int age){
}

当使用 raw 中的 JSON 进行传输时，Springboot 可以通过 @RequestBody 进行接收。此接收方式只能够接收 1 个对象，这个对象可以包含多个对象，要是需要同时传递两个对象，则需要使用 @RequestPart 进行接收。

@PostMappping("\test")
public String test(@RequestBody Map<String,String> data){
}

当需要传输文件时，可以选择 multipart/form-data 格式，Springboot 需要使用 MultipartFile 进行接收。

@PostMappping("\test")
public String test(MultipartFile multipartFile, String data){
}

环境变量和集合

在 Postman 中，环境变量和集合是两个非常有用的功能，它们可以帮助你更好地管理请求和测试。下面我将简要介绍一下这两个功能：环境变量和集合。

环境变量

环境变量允许你在不同的环境之间轻松切换请求参数。比如，你可以设置一个开发环境和一个生产环境，并为每个环境设置不同的变量值。

  • 创建环境变量：在 Postman 的右上角点击眼睛图标，选择 “Manage Environments”。在弹出的窗口中，你可以创建新的环境并添加变量。

  • 使用环境变量：在请求的 URL、请求头、查询参数、表单数据等地方使用双括号语法（例如：）来引用环境变量。当你发送请求时，Postman 会自动替换这些变量为对应的值。

在环境变量中分成了全局环境变量和环境变量：

  • 全局变量（Global Variables）：全局变量是在 Postman 的整个应用程序中可用的变量，可以被所有的集合和请求使用。全局变量适用于那些在不同环境下都需要使用的通用数据。
  • 环境变量（Environment Variables）：环境变量是针对特定环境而设定的变量，只能在特定环境中使用。每个环境都可以设置自己的环境变量，这些变量将覆盖或补充全局变量。

全局变量和环境变量都可以用来存储和传递数据，比如 API 的基础 URL、身份验证令牌等。当一个请求被执行时，Postman 会首先检查环境变量，如果某个变量在环境中被定义，则使用环境变量的值。如果该变量在环境中未定义，Postman 会继续检查全局变量，并使用全局变量的值。如果变量既没有在环境中定义也没有在全局中定义，则使用默认值（如果有的话）。在实际使用中，全局变量适合那些在不同环境下都需要使用的通用数据，例如开发和生产环境共享的 API 基础 URL。而环境变量则适用于特定环境中的数据，如不同测试环境或不同用户身份验证令牌。

集合

集合是一种组织和管理请求的方式。你可以将相关的请求分组到一个集合中，以便更好地组织和执行。

  • 创建集合：在 Postman 的左上角点击 “New” 按钮，选择 “Collection”。然后，为集合命名，并可以选择一个文件夹进行组织。

  • 添加请求到集合：在集合中，你可以添加请求并设置请求的各项详细信息，如 URL、请求方法、参数等。

  • 执行集合：你可以选择执行整个集合或者只执行其中的某些请求。执行集合会依次发送每个请求，并记录响应结果和测试结果等信息。

测试和调试

在 Postman 中，你可以使用测试和调试功能来验证 API 响应的正确性和进行故障排查。下面我将简要介绍一下这两个功能：测试和调试。

测试

测试是在请求发送后自动运行的代码块，用于验证 API 响应的正确性。你可以使用 JavaScript 编写测试脚本，并将其添加到 Postman 请求中。

  • 添加测试脚本：在请求编辑器中的 “Tests” 选项卡中，你可以编写 JavaScript 代码来验证响应。常见的验证包括状态码、响应体内容、响应时间等。

  • 运行测试：当你发送请求时，Postman 会自动运行测试脚本，并在请求结果中显示测试结果。如果测试失败，你可以检查错误信息并进行修正。

调试

调试功能可以帮助你快速定位并解决 API 请求的问题。你可以逐步执行请求，查看请求和响应的详细信息，并使用断点进行调试。

  • 设置断点：在请求编辑器中的 “Tests” 选项卡中，你可以通过在代码中添加 debugger; 语句来设置断点。当代码执行到断点处时，请求将被暂停。

  • 查看变量和日志：在断点处，你可以查看请求和响应的详细信息，以及当前变量的值。你还可以使用 console.log() 函数输出调试信息。

高级特性和扩展

高级特性

  1. 脚本： Postman提供了预请求脚本（Pre-request Script）和测试脚本（Test Script）的功能，你可以使用JavaScript编写脚本来处理请求前的操作和响应后的验证。这使得你可以执行动态操作、处理变量、进行数据转换等。
  2. 集合： 集合（Collection）是一组关联的请求的容器。你可以将相关的请求分组到一个集合中，并使用环境变量、全局变量和脚本等功能来管理和执行集合中的请求。
  3. 自动化： Postman支持自动化测试和持续集成，你可以使用Postman的命令行工具或与CI/CD工具集成，以便在构建和部署过程中自动运行集合并生成报告。
  4. Mock 服务器： Postman提供了Mock服务器功能，你可以创建一个虚拟的API服务器来模拟API的行为。这对于前端开发、单元测试和团队协作非常有用。
  5. 监控： Postman提供了API监控功能，可以周期性地发送请求并监视API的性能和可用性。你可以设置警报条件，并通过电子邮件或Webhook接收通知。

扩展

Postman的功能可以通过扩展进行进一步扩展。Postman提供了一个丰富的插件系统，你可以安装和使用各种扩展，如自定义认证、数据生成、OpenAPI支持等。

最佳实践

下面是一些Postman的最佳实践和案例分析：

  1. 使用环境变量： 使用环境变量可以使你的请求更加灵活和可重用。你可以在环境中定义变量，并在请求中引用这些变量。这使得你可以轻松地在不同的环境中切换（如本地开发、测试和生产环境），而不需要手动更改请求的URL或参数。
  2. 使用集合和目录： 将相关请求分组并放置在集合和目录中可以使你的请求更加有条理和易于管理。你可以为每个集合和目录指定描述和标签，以便快速搜索和识别。
  3. 编写测试脚本： 编写测试脚本可以帮助你验证API响应的正确性和一致性。你可以使用JavaScript编写测试脚本来检查状态码、检查响应体内容、比较变量值等。
  4. 自动化测试： 自动化测试可以帮助你自动执行测试并生成测试报告。你可以使用Postman的命令行工具或与CI/CD工具集成，以便在构建和部署过程中自动运行集合并生成报告。
  5. 使用Mock服务器： 使用Mock服务器可以模拟API的行为，使你可以在API未完全实现之前开发前端。你可以使用Postman的Mock服务器功能来创建虚拟API服务器，并为每个端点指定响应。
  6. 使用OAuth2认证： 使用OAuth2认证可以帮助你保护API，并控制对API的访问。你可以使用Postman的OAuth2认证功能来设置授权令牌，并将其添加到请求中。

这些最佳实践和案例分析可以帮助你更好地利用Postman的功能，提高API的可靠性和可维护性。无论你是单独开发还是与团队合作，这些实践都很有用。

结论和展望

结论

总体而言，Postman是一个功能强大且易于使用的API测试和开发工具。它提供了丰富的功能，包括发送HTTP请求、管理环境变量、编写测试脚本、自动化测试等。通过使用Postman，开发人员可以更高效地构建和管理API，并确保其可靠性和一致性。

展望

随着云计算和微服务架构的普及，API在现代软件开发中扮演着越来越重要的角色。未来，我们可以期待Postman继续发展和创新，以满足不断变化的API测试和开发需求。

以下是一些可能的发展方向：

  1. 更多的集成和扩展：Postman已经与许多常用的工具和平台（如Jenkins、GitHub、Slack等）进行了集成，以便更好地支持CI/CD和团队协作。未来，我们可以期待更多的集成和扩展，以满足不同开发团队的需求。
  2. 更强大的自动化测试功能：自动化测试对于保证API质量和稳定性至关重要。未来，Postman可以进一步增强其自动化测试功能，提供更多的测试类型和灵活性，以满足不同场景下的测试需求。
  3. 更好的性能测试支持：性能是API的一个重要指标，特别是在高并发和大流量的情况下。未来，我们可以期待Postman提供更好的性能测试支持，帮助开发人员评估API的性能并进行优化。
  4. 更友好的用户界面和体验：虽然Postman已经提供了直观的用户界面和易于使用的功能，但随着用户需求的增长，我们可以期待更多的改进和优化，以提供更好的用户体验。

综上所述，Postman作为一款出色的API测试和开发工具，有着广阔的发展前景。无论是在单独开发还是团队协作中，Postman都将继续在简化、加速和提升API开发过程中发挥重要作用。
