泛型

Harvey Xie 2023-11-25 11:51:14
Categories： Java Tags： Generics

《编程思想》中提到泛型，其描述是：“有许多原因促成了泛型的出现，而最引人注目的一个原因，就是为了创造容器类”。泛型属于一种编译时类型安全检测机制，避免在运行时强转类型时报类型转化异常。从 JDK 1.5时，Java 引入了泛型的概念，泛型即“参数化类型”，使代码可以应用于多种类型。

所谓的“参数化类型”其实可以拆成两句话来理解：它是一种类型，这种类型是参数化的。在 Java 语言中，像 int、long、double等 ，这些属于基本数据类型，String、对象等属于引用数据类型，所以泛型也和它们一样是一种类型。方法的参数有形参和实参，形参定义在方法上，实参由实际的调用者传递给方法，那么参数化指的就是泛型可以像形参那样定义在方法上，具体使用的时候由调用者来决定。

假设没有泛型

在 JDK 1.5 之前没有泛型的时候，我们可以向容器类中添加任意类型的对象。这就存在安全隐患，比如向容器添加字符串类型的数据，但是不小心强转成了其他类型，此时就会报 ClassCastException 错误。

public static void main(String[] args) {
    List list = new ArrayList();
    list.add(1);
    list.add(2);
    list.add("abc");
    list.add(3);
    for (int i = 0; i < list.size(); i++) {
        System.out.println(((Integer) list.get(i)));
    }
}

消除上述安全隐患的方法是让添加到容器类中的对象只有一种类型。此时，泛型的出现就可以很好的规避这个问题：

public static void main(String[] args) {
    List<Integer> list = new ArrayList<>();
    list.add(1);
    list.add(2);
    //list.add("abc"); 在编译的时候，就会报错。
    list.add(3);
    for (int i = 0; i < list.size(); i++) {
        System.out.println(((Integer) list.get(i)));
    }
}

第 5 行代码想要向 Integer 类型的容器中添加一个字符串类型的时候，编译器就会提示错误。泛型将原本发生在运行时的错误提前到了编译期，让开发者可以更快的发现错误，提高代码的安全性。

泛型擦除

泛型是 JDK 1.5 版本引入的概念。为了向前保持兼容，泛型只会存在编译阶段，当进行运行时，与泛型相关的信息会被虚拟机擦除。该过程我们称为泛型擦除。泛型擦除的目的之一是为了实现兼容。

基本使用

泛型可以使用在方法、类和接口上。通常，约定熟成使用如下的泛型标识符来表示特殊的含义：

标识符 含义 作用
E Element 在容器类中使用，表示容器类中的元素
T Type Class 类
K Key 键
V Value 值
? 不确定的类型 ——

泛型类不支持基本数据类型。

泛型方法

泛型方法需要定义在返回值之前，形如：

public <泛型标识,泛型标识,泛型标识,...> void fun() {
}

泛型方法在调用该方法时需要传递具体的泛型参数。泛型方法可以独立与泛型类而存在，也就是说一个泛型方法可以在一个不是泛型类中。

泛型类

泛型类的泛型需要定义在类名后，形如：

class className <泛型标识,泛型标识,泛型标识,...> {
    private 泛型标识 变量名;
}

如下是使用示例：

class GenericClass<T> {
    private T t;

    public GenericClass(T t) {
        this.t = t;
    }

    public static void main(String[] args) {
        GenericClass<String> stringGenericClass = new GenericClass<>("Hello Generic");
        GenericClass<Integer> integerGenericClass = new GenericClass<>(100);
    }
}

泛型类在继承关系中需要注意以下几点：

  1. 子类的类型参数必须与父类的类型参数相同或是其子类型；

    class MyBaseClass<T> {
	
}

class MyClass1<T> extends MyBaseClass<T> {

}

class MyClass2<T extends MyBaseClass> extends MyBaseClass<T> {

}

  2. 子类可以增加自己的类型参数，但是不能减少或修改父类的类型参数；

    class MyBaseClass<T,U> {

}

class MyClass1<T> extends MyBaseClass<T> {
	//此处会编译报错：Incorrect number of arguments for type MyBaseClass<T,U>; 
    // it cannot be parameterized with arguments <T>
}

class MyClass2<U> extends MyBaseClass<T> {
	//此处会编译报错：T cannot be resolved to a type
}

  3. 子类可以覆盖父类的泛型方法，但是覆盖后的方法的类型参数必须与父类的方法类型参数相同；

    当覆盖父类的泛型方法时，子类的中方法必须与父类方法具有相同的方法签名，包括方法名、参数类型和数量。因此，子类不可以通过改变泛型参数的类型范围来覆写父类的泛型方法。

    class MyBaseClass {
	public <T> void print(T t) {}
}

class ChildClass extends ParentClass {
	@Override
	public <T> void print(T t){}
}

  4. 泛型继承关系不能用于静态方法或是静态变量，因为它们在编译时就已被解析；

  5. 在使用泛型继承关系时，因为 Java 泛型是通过类型擦除实现的，因此在编译时会将泛型类型擦除为其上限类型，从而可能导致一些类型安全问题。

泛型接口

泛型接口和泛型类一样，泛型接口定义在接口名后，形如：

interface className <泛型标识,泛型标识,泛型标识,...> {
    泛型标识 fun();
}

在泛型接口中，没有明确指定类型时，默认是 Object 类型。实现类不是泛型类，接口应该明确数据类型；实现类也是泛型类，实现类和接口的泛型需要保持一致。

interface Action<T> {
	T get();
}

//默认是 Object 类型
class MyClass1 implements Action {
	@Override
	public Object get() {
		return null;
	}
}

//实现类不是泛型类，接口要明确数据类型
class MyClass2 implements Action<String> {
	@Override
	public Object get() {
		return null;
	}
}

//实现类也是泛型类，实现类和接口的泛型需要保持一致
class MyClass3<T> implements Action<T> {
	@Override
	public T get() {
		return null;
	}
}

边界

无界通配符

当泛型的类型不明确时，可以使用无界通配符 ? 声明泛型类型。使用无界通配符通配符声明的泛型，由于不明确其具体类型，因此只能读操作，不可以写操作，且读到对象是 Object 类型。

public class Main {
    public static void main(String[] args) {
        List<String> strings = new ArrayList<>();
        strings.add("a");
        strings.add("b");
        strings.add("c");
        
        List<Integer> integers = new ArrayList<>();
        integers.add(1);
        integers.add(2);
        integers.add(3);

        Main main = new Main();
        main.addToList(strings);
        main.addToList(integers);
    }

    public void addToList(List<?> list) {
        // list.add("a"); 此处编译报错
        // list.add(1); 此处编译也会报错
        for (Object o : list) {
            // 只能读，不能写
        }
    }
}

上界

使用 extends 关键字对泛型进行上界的限定。对泛型的类型进行上界限定后，限定传递的参数只能是实参类型或者泛型的子类型。无界通配符默认是 Object 类型，因此无界通配符也可以认为是最大的上界。做了上界限定后，因为不知道其具体的子类型，其子类型可能存在无数多个，因此也只能进行读操作，而无法对其写操作，读到类型为泛型上界的类型。

public class Main {
    public void start(String[] args) {
        List<DragonLi> dragonLis = new ArrayList<>();
        dragonLis.add(new DragonLi());
        dragonLis.add(new DragonLi());

        List<Ragdoll> ragdolls = new ArrayList<>();
        ragdolls.add(new Ragdoll());
        ragdolls.add(new Ragdoll());

        addToList(dragonLis);
        addToList(ragdolls);
    }

    public void addToList(List<? extends Cat> animals) {
        for (Object o : animals) {
            // 只能读，不能写
        }
        // animals.add(new Ragdoll()) 会报错
    }

    class Animal {
    }

    class Cat extends Animal {
    }

    class DragonLi extends Cat {
    }

    class Ragdoll extends Cat {
    }
}

下界

使用 supers 关键字对泛型进行下界的限定。对泛型的类型进行下界限定后，限定传递参数只能是实参类型或者泛型的父类型。做了下界限定的泛型可以进行读和写操作，读到的是一个 Object 类型。

public class Main {
    public void start(String[] args) {
        List<Cat> cats = new ArrayList<>();
        cats.add(new Cat());
        cats.add(new DragonLi()); // 里氏替换法
        cats.add(new Ragdoll()); // 里氏替换法

        List<Dog> dogs = new ArrayList<>();
        dogs.add(new Dog());
        dogs.add(new Golden());
        dogs.add(new Labrador());

        addToList(cats);
        // addToList(dogs); 无法传递，因为上界限定只能是 Cat 或是 Cat 的子类型
    }

    public void addToList(List<? super Cat> likeCat) {
        for (Object cat : likeCat) {
            // 可以读，但是读到是一个 Object 类型
        }
        likeCat.add(new Cat()); // 可以进行写操作
    }

    class Animal {
    }

    class Cat extends Animal {
    }

    class Dog extends Animal {
    }

    class DragonLi extends Cat {
    }

    class Ragdoll extends Cat {
    }

    class Golden extends Dog {
    }

    class Labrador extends Dog {
    }
}

里氏替换法则告诉我们，父类出现的地方可以使用子类代替（或者也可以理解成初始化子类之前必须先初始化父类），因此向其插入子类是安全的。
