反射

Harvey Xie 2023-11-25 11:51:14
Categories： Java Tags： Reflection

反射是指在运行时动态地获取类的信息以及使用类的信息的机制。通过反射机制，可以在程序运行时检查类、接口、字段和方法，甚至在运行时实例化对象、调用方法、访问或修改字段，而这些操作通常是在编译时无法确定的。在 Java 中，每个类都对应一个 Class 对象，该对象包含了与类有关的元数据信息，如类的名称、字段、方法、构造函数等。通过反射可以获取并操作这些元数据信息，从而实现对类的动态处理和操作。

反射的作用

通过反射，可以做到以下几个方面的操作：

  1. 获取 Class 对象：可以通过类的完全限定名或对象实例来获取对应的 Class 对象。
  2. 获取类的信息：可以获取类的字段、方法、构造函数等信息，以及类上的注解等元数据。
  3. 实例化对象：可以通过 Class 对象实例化对象，而不需要直接使用 new 关键字。
  4. 调用方法：可以动态调用类的方法，包括公有、私有方法，静态方法等。
  5. 访问和修改字段：可以读取和设置对象的字段，包括公有、私有字段，静态字段等。

反射提供了一种非常灵活、强大的机制，能够使程序员在编写程序时更加灵活地处理类和对象。然而，反射也可能会带来性能上的损失，因此在使用反射时需要谨慎考虑性能和安全方面的问题。

反射的使用

获取字节码对象

在使用反射前，需要先获取到字节码对象。获取字节码对象的方式有如下三种：

  1. 已知类的情况下，使用 .class 语法获取对象的 Class 对象；

  2. 已知包名和类名的情况下，使用 Class.forName() 或者 ClassLoader 动态获取 Class 对象；

  3. 已有实例化对象，通过 getClass() 获取 Class 对象；

获取类信息

通过 Class 对象可以获取类的字段、方法、构造函数等信息，以及类上的注解等元数据。

实例化对象

通过 Class 对象可以实例化对象，可以通过无参构造函数或带参构造函数来创建对象。

调用方法

通过 Method 对象可以调用类的方法，包括公有、私有方法，静态方法等。

访问和修改字段

通过Field对象可以读取和设置对象的字段，包括公有、私有字段，静态字段等。

动态代理

在 Java 中，动态代理的实现通常依赖于反射机制。动态代理是通过在运行时创建代理对象来实现的，而这些代理对象可以动态地拦截方法调用并执行额外的逻辑。为了实现这一功能，需要在运行时获取目标类的信息，包括方法和参数等。而反射机制正是提供了一种在运行时获取、检查和操作类的方法、字段、构造函数等的能力。

动态代理的实现方式有两种：基于接口的动态代理和基于类的动态代理。

  • 在基于接口的动态代理中，Java 的 Proxy 类使用了反射机制。它提供了一个静态方法newProxyInstance()，该方法接受一个类加载器、一组接口和一个 InvocationHandler 作为参数，然后在运行时生成一个代理类的实例。这个代理类实现了指定接口，并且通过 InvocationHandler 的实现类来处理方法的调用。当代理对象的方法被调用时，实际上是通过反射机制将方法调用委托给 InvocationHandler 的invoke()方法。

  • 在基于类的动态代理中，一般使用第三方库，比如cglib。cglib 也是通过反射机制来实现动态代理。它创建一个目标类的子类，并重写父类中的方法来拦截方法调用。在拦截器中，同样使用反射机制来执行额外的逻辑。

基于接口的动态代理

基于接口的动态代理需要目标类实现一个或多个接口，在运行时创建一个代理类的实例，该实例实现了目标类所实现的接口，并且可以拦截方法调用并执行额外的逻辑。Java 中提供了一个 Proxy 类，用于创建基于接口的动态代理对象。

基于类的动态代理

基于类的动态代理不需要目标类实现接口，相反它创建一个目标类的子类，并重写父类中的方法来拦截方法调用并执行额外的逻辑。Java 中提供了一个 cglib 库，用于创建基于类的动态代理对象。
