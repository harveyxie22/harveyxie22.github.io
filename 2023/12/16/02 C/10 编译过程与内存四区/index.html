编译过程与内存四区

Harvey Xie 2023-12-16 17:00:00
Categories： C Tags： 语言基础

编译过程

C 语言的编译过程一般包括以下几个步骤：

  1. 预处理（Preprocessing）：在源代码被编译前，预处理器会对源代码进行处理，例如替换宏定义、展开头文件等操作。预处理器生成的代码通常以 .i 或 .ii 为扩展名，称为预处理后的源代码。
  2. 编译（Compilation）：编译器将预处理后的源代码翻译成汇编语言，生成一个或多个汇编代码文件（通常以 .s 为扩展名），每个汇编代码文件对应一个源代码文件。
  3. 汇编（Assembly）：汇编器将汇编代码转化成机器指令，生成一个或多个目标文件（通常以 .o 为扩展名），每个目标文件对应一个汇编代码文件。
  4. 链接（Linking）：链接器将所有的目标文件和库文件合并成一个可执行文件。链接器还会对目标文件中的符号进行解析和重定位，以生成最终的可执行文件。

不同的编译器可能采用不同的实现方式，但基本流程大致相同。在编辑器中进行编译时，这些步骤通常被自动化处理，编译器会自动按照这些步骤对源代码进行处理，并生成可执行文件。

命令

C语言的每一个编译过程使用的命令是不一样的。

预处理阶段

gcc -E HelloWorld.c -o HelloWorld.i

参数-E只进行预处理。参数-o指定输出文件名。

编译阶段

gcc -S HelloWorld.i -o HelloWorld.s

参数-S只进行预处理和编译。参数-o指定输出文件名。

汇编阶段

gcc -c HelloWorld.s -o HelloWorld.o

参数-c只进行预处理、编译和汇编。参数-o指定输出文件名。

链接阶段

gcc HelloWorld.o -o HelloWorld

实际开发

实际的开发中，我们可以使用：

gcc HelloWorld.c -o HelloWorld

一步到位完成上述四个阶段的全部操作。

内存四区

C 语言经过预处理、编译、汇编和链接四个步骤后可以生成一个可执行文件。在操作系统下，可以查询到该执行文件的基本情况。假设存在一个 HelloWorld.c 的文件，那么：

gcc HelloWorld.c -o helloworld.exe

得到一个可执行文件后，执行：

file helloworld.exe
helloworld.exe: ELF 64-bit LSB shared object, x86-64, version 1 (SYSV), dynamically
linked, interpreter /lib64/ld-linux-x86-64.so.2, 
BuildID[sha1]=e2c48a9abae67a215e8c98a7765a67792c0bd1b1, for GNU/Linux 3.2.0, not stripped

最后，通过 size 查看可执行文件的结构情况：

size helloworld.exe
   text	   data	    bss	    dec	    hex	filename
   1623	    600	      8	   2231	    8b7	helloworld.exe

上述字段的含义如下表所示：

字段 含义
text 代码区
data 静态数据或全局初始化数据区
bss 未初始化数据区
dec 十进制总和
hex 十六进制总和
filename 文件名

也就是说，当一个可执行文件在没有加载到内存之前，已经被分成了三段信息：代码区（text）、数据区（data）和未初始化数据区（bss）。

程序在加载到内存前，代码区和全局区（data和bss）的大小是固定的，程序运行期间不能够改变。然后当程序可运行的时候，系统会把程序加载到内存，除了代码区、数据区和未初始化的数据区以外，还会额外新增栈区和堆区。

代码区用于存放 CPU 执行的机器指令。通常代码区是可共享的（即另外的执行程序可以调用它），使其可共享的目的是对于频繁被执行的程序，只需要在内存中有一份代码即可。代码区通常是只读的，使其只读的原因是防止程序意外地修改了它的指令。另外，代码区还规划了局部变量的相关信息。简单来说：程序员所写的所有代码都会放入到代码区中，代码区的特点是共享和只读。

全局区中主要存放的数据有：全局变量、静态变量、常量（如字符串常量）。全局区的叫法有很多：全局区、静态区、数据区、全局静态区、静态全局区。这部分可以细分为 data 区和 bss 区。

  • data区：主要存放的是已经初始化的全局变量、静态变量和常量；
  • bss区：主要存放的是未初始化的全局变量、静态变量，这些未初始化的数据在程序执行前会自动被系统初始化为0或者NULL；
  • 常量区：是全局区中划分的一个小区域，里面存放的是常量，如const修饰的全局变量、字符串常量等。

栈区是一种先进后出的内存结构，由编译器自动分配释放，存放函数的参数值、返回值、局部变量等。在程序运行过程中实时加载和释放，因此，局部变量的生存周期为申请到释放该段栈空间。

堆区是一个大容器，它的容量要远远大于栈，但没有栈那样先进后出的顺序。用于动态内存分配。堆在内存中位于BSS 区和栈区之间。一般由程序员分配和释放，若程序员不释放，程序结束时由操作系统回收。

作用域与内存

类型 作用域 生命周期 存储位置

堆内存

堆的存储空间要比栈大很多，当数据很大的时候，就可以考虑把数据存放到堆中。向堆申请一块内存空间可以使用函数malloc()，当空间使用完毕后，可以调用 free() 释放该空间。

  1. malloc

    该函数的声明void *malloc(size_t size);。作用是在内存的动态存储区（堆区）中分配一块长度为size字节的连续区域，用来存放类型说明符指定的类型。分配的内存空间内容不确定，一般使用memset初始化。若申请成功返回值是分配空间的起始地址，若失败返回NULL。

  2. free

    该函数的声明void free(void *ptr)。作用是释放ptr所指向的一块内存空间，ptr是一个任意类型的指针变量，指向被释放区域的首地址。对同一内存空间多次释放会出错。注意，被释放区应是由malloc函数所分配的区域。

以下是一个在堆内存中申请内存的例子：

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main()
{
    // 1.向堆申请一块10个大小的int类型的空间
    int *array = (int *)malloc(10 * sizeof(int));

    if (array == NULL)
    {
        printf("ERROR!");
        return -1;
    }

    // 2.把申请到的内存空间清0
    memset(array, 0, sizeof(int) * 10);

    // 3.向申请到的数据添加数据
    for (size_t i = 0; i < 10; i++)
    {
        array[i] = i;
    }

    // 4.打印数据
    for (size_t i = 0; i < 10; i++)
    {
        printf("%d\n", array[i]);
    }

    // 5.释放申请的空间
    free(array);

    return 0;
}

内存分区与代码分析

返回栈区地址

当函数返回栈区中的地址时，会报错。这是因为栈区当函数执行完毕后会被释放。

int *fun(){
    int a = 10;
    return &a;
}

int main()
{
    int *p = NULL;
    p = fun();
    *p = 100;
}

返回data区地址

#include <stdio.h>

int *fun()
{
	static int a = 10;
	return &a; //函数调用完毕，a不释放
}

int main(int argc, char *argv[])
{
	int *p = NULL;
	p = fun();
	*p = 100; //ok
	printf("*p = %d\n", *p);

	return 0;
}

返回堆区地址

#include <stdio.h>
#include <stdlib.h>

int *fun()
{
	int *tmp = NULL;
	tmp = (int *)malloc(sizeof(int));
	*tmp = 100;
	return tmp;//返回堆区地址，函数调用完毕，不释放
}

int main(int argc, char *argv[])
{
	int *p = NULL;
	p = fun();
	printf("*p = %d\n", *p);//ok

	//堆区空间，使用完毕，手动释放
	if (p != NULL)
	{
		free(p);
		p = NULL;
	}

	return 0;
}

值传递

#include <stdio.h>
#include <stdlib.h>

void fun1(int *tmp)
{
    *tmp = 100;
}

void fun2(int *tmp)
{
    tmp = (int *)malloc(sizeof(int));
    *tmp = 100;
}

int main(int argc, char *argv[])
{
    int *p = NULL;
    p = (int *)malloc(sizeof(int));

    fun1(p);                 //值传递
    printf("*p = %d\n", *p); // ok，*p为100

    int *q = NULL;
    fun2(q);                 //值传递，形参修改不会影响实参
    //printf("*p = %d\n", *q); // ERROR，操作空指针指向的内存

    return 0;
}

总结

在理解 C/C++ 内存分区时，常会碰到如下术语：数据区，堆，栈，静态区，常量区，全局区，字符串常量区，文字常量区，代码区等等，初学者被搞得云里雾里。在这里，尝试捋清楚以上分区的关系：

  • 数据区包括：堆，栈，全局/静态存储区。
  • 全局/静态存储区包括：常量区，全局区、静态区。
  • 常量区包括：字符串常量区、常变量区。
  • 代码区：存放程序编译后的二进制代码，不可寻址区。

可以说，C/C++ 内存分区其实只有两个，即代码区和数据区。
