单例模式

Harvey Xie 2023-12-09 15:30:00
Categories： Java Tags： 设计模式

在 Java 中，单例模式（Singleton Pattern）是非常常见的设计模式之一。它确保一个类只有一个实例，并提供全局访问点来获取该实例。

在 Java 中，有四种常见的单例模式：

  1. 懒汉式单例模式（Lazy Initialization）：这种方式在首次使用时才创建实例。线程不安全，需要考虑线程同步问题；
  2. 饿汉式单例模式（Eager Initialization）：这种方式在类加载时就创建了实例。线程安全，但可能会造成不必要的实例化；
  3. 双重检查锁定单例模式（Double-Checked Locking）：这种方式在懒汉式的基础上增加了线程安全的检查；
  4. 静态内部类单例模式（Static Inner Class）：这种方式使用静态内部类来延迟初始化实例，并且实现了线程安全。

单例模式

懒汉式单例模式

public class Singleton {
    private static Singleton instance;

    private Singleton() {
        // 私有化构造函数
    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

饿汉式单例模式

public class Singleton {
    private static final Singleton instance = new Singleton();

    private Singleton() {
        // 私有化构造函数
    }

    public static Singleton getInstance() {
        return instance;
    }
}

双重检查锁定单例模式

public class Singleton {
    private volatile static Singleton instance;

    private Singleton() {
        // 私有化构造函数
    }

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

静态内部类单例模式

public class Singleton {
    private Singleton() {
        // 私有化构造函数
    }

    private static class Holder {
        private static final Singleton instance = new Singleton();
    }

    public static Singleton getInstance() {
        return Holder.instance;
    }
}

优缺点

特点\模式 懒汉式单例模式 饿汉式单例模式 双重检查锁定单例模式 静态内部类单例模式
优点 延迟初始化，只有在需要时才创建实例。 1. 在类加载时就创建实例，保证了线程安全；
2. 获取实例的性能较好。		 1. 延迟初始化，只有在需要时才创建实例；
2. 使用双重检查锁定机制，在保证线程安全的同时，减少了同步操作次数，提高了性能。		 1. 延迟初始化，只有在需要时才创建实例；
2. 利用类加载机制，保证了线程安全；
3. 性能较好，获取实例的开销很小。
缺点 1. 线程不安全，多线程环境下可能会出现并发访问问题，需要进行线程同步处理；
2. 每次获取实例都需要进行同步，性能较差。		 不管是否使用，都会实例化对象，可能造成不必要的资源浪费。 1. 代码相对复杂，容易出错；
2. 对于早期的JDK版本，双重检查锁定可能存在问题，需要使用volatile关键字来修饰实例变量。		 无法传递参数给私有化构造函数。

根据上述描述，可以总结如下：

  • 如果不考虑线程安全和性能问题，可选择懒汉式单例模式。
  • 如果需要保证线程安全且能够在类加载时就创建实例，可选择饿汉式单例模式。
  • 如果需要延迟初始化、保证线程安全且性能较好，可选择双重检查锁定单例模式。
  • 如果需要延迟初始化、保证线程安全且性能最优，可选择静态内部类单例模式。

经典案例

日志记录器

在大多数应用程序中，日志记录是一个重要的组件。通过使用单例模式的日志记录器，可以确保在整个应用程序中只有一个日志实例，并且可以方便地记录日志信息。

public class Logger {
    private static Logger instance;
    
    private Logger() {
        // 私有化构造函数
    }
    
    public static synchronized Logger getInstance() {
        if (instance == null) {
            instance = new Logger();
        }
        return instance;
    }
    
    public void log(String message) {
        System.out.println("Logging: " + message);
    }
}
