宏定义与常量

Harvey Xie 2023-12-10 11:43:00
Categories： C Tags： 语言基础

宏

定义

在C语言中，宏定义分两种：无参数宏定义和有参数宏定义。宏定义是常见的预处理指令，使用“标识符”来表示“替换列表”中的内容，标识符称为宏名。在编译过程中的预处理阶段，编译器会把源代码中的宏定义替换成列表中内容。

无参宏定义

无参宏定义的格式为：

#define 标识符 替换列表

替换列表可以是数值常量、字符常量、字符串常量等。例如，需要宏定义一个圆周率π，可以写成：

#define PI 3.1415926

使用宏定义的时候，需要注意#define最好另起一行写，最后不需要添加分号;，若添加分号的话，虽然不会提示错误，但是在预处理阶段的时候会造成错误。例如：

#define SIZE 10;

在预处理的时候假如存在int a[SIZE];，那么替换后就变成了：

int a[10;];

很明显多了一个分号，此时就会造成代码运行的过程中出现错误。

有参宏定义

有参宏定义的格式为：

#define 标识符(参数1,参数2,参数3,...)替换列表

例如需要使用有参的宏定义来比较两个参数最大的值，就是：

#define MAX(a,b) ((a)>(b)?(a):(b))

常量

定义

常量有时候也被称为字面量。常量在程序执行期间不会改变，因此常量也可以称为只读变量。常量可以是 C 语言中任何基本数据类型。在 C 语言中，可以使用 const 来定义一个常量。

const int var = 5;

宏定义和常量的区别

宏定义和使用 const 常量的两者区别是：

  1. 编译器处理方式不同。宏定义是在预处理阶段展开，const 常量是在编译运行阶段使用；
  2. 类型和安全检查不同。#define 宏没有类型，不做任何类型检查，仅仅是展开。const 常量有具体的类型，在编译阶段会执行类型检查；
  3. 存储方式不同。#define 宏仅仅是展开，有多少地方使用，就展开多少次，不会分配内存。宏定义不分配内存，变量定义分配内存。const定义常量会分配内存。

定义与声明

在 C 语言中，变量的定义和声明是两个概念。

声明（Declaration） 是指在程序中引入一个标识符（变量名），告诉编译器该标识符的存在、类型以及作用域。通过声明，我们可以让编译器知道某个变量将在后续的代码中被使用。声明不会分配内存空间。

定义（Definition） 是指在程序中为一个标识符（变量名）分配内存空间，并且可以给该标识符赋初值。变量的定义包括了变量的声明，同时也进行了内存分配。

下面是变量的声明与定义的示例：

// 只有声明，没有定义
extern int x;
extern float y;

int main() {
    // 定义和声明
    int a;  // 定义了一个名为 a 的整型变量
    float b = 3.14;  // 定义了一个名为 b 的浮点型变量，并初始化为 3.14
    char c, d, e;  // 定义了三个名为 c、d、e 的字符型变量

    // 声明，没有定义
    extern double z;

    return 0;
}

在上述示例中，x 和 y 是在 main 函数外部声明的全局变量，a、b、c、d、e 是在 main 函数内部定义的局部变量。z 是通过 extern 关键字声明的全局变量，它在 main 函数内部没有定义。

需要注意的是，C 语言要求在使用变量之前必须先进行声明或定义。如果只有声明而没有定义，那么该变量的内存空间并没有被分配。而只有定义而没有声明，则默认为隐式声明。

总结可得如下几点：

  1. 带有初始化值的，不管是否有 extern 关键字，都被认为是定义；
  2. 凡是带有 {} 的函数一定是定义，否则就是声明；
  3. 除非有 extern 关键字，否则都是变量的定义。

作用域

C 语言中的作用域是指一个标识符（变量、函数等）在程序中可见的范围。C 语言中有以下几种作用域：

  1. 块作用域（Block Scope）：在一个代码块（由一对花括号 {} 包围的代码）内定义的变量具有块作用域，只能在该块内部访问。代码块可以是函数、循环或条件语句中的花括号内部。
  2. 函数作用域（Function Scope）：在函数内部定义的变量具有函数作用域，只能在函数内部访问。函数参数和函数内部定义的变量都具有函数作用域。
  3. 文件作用域（File Scope）：在函数外部定义的变量具有文件作用域，可以在整个文件内的任何位置访问。文件作用域的变量在函数之外声明，通常用于全局变量。
  4. 函数原型作用域（Function Prototype Scope）：在函数原型中声明的形式参数具有函数原型作用域，它们只在函数原型中起作用。

需要注意的是，C 语言没有块级作用域，即代码块内部定义的变量在块外部仍然可见。此外，C99 标准引入了块作用域的变量声明和初始化的特性，可以在代码块中任意位置声明和初始化变量。

作用域规则允许在不同的作用域中使用相同名称的变量，但每个作用域内的变量是独立的，不会相互影响。当存在嵌套作用域时，内部作用域的变量将隐藏外部作用域中具有相同名称的变量。

对于全局变量，可以使用关键字 extern 来在函数内部引用它们。这样，函数内部就可以访问全局变量，但不能重新定义它。

局部变量和全局变量

按照变量的位置，可以分成：局部变量和全局变量。

局部变量也叫做 auto 自动变量（auto可省略不写），通常在代码块（一对大括号内）内部定义的变量都是自动变量。自动变量的特点是，在一个函数内定义，则只在该函数范围内有效。在复合语句中定义，只在复合语句中有效。但是，随着函数或复合语句的执行结束，局部变量的声明周期也随着结束。若局部变量初始化时没有赋值，则内容随机。被 static 关键字修饰的局部变量是一个静态局部变量。静态局部变量只初始化一次，但是可以多次赋值，其作用域在定义的函数内有效，生命周期和程序运行周期相同。静态局部变量若未赋以初值，则由系统自动赋值：数值型变量自动赋初值0，字符型变量赋空字符。

全局变量指的是在函数外定义，可以被本文件以及其他文件的函数所使用的变量。当需要被其他文件使用的时候，需要添加 extern 声明。全局变量的声明周期与程序运行相同，不问的文件的全局变量不可重名。被 static 关键字的全局变量称为静态全局变量。静态全局变量定义在函数外，作用范围被限制在所定义的文件中，不同文件静态全局变量可以重名，但作用域不冲突。静态全局变量的生命周期和程序运行周期一样，同时静态全局变量的值只初始化一次。

在 C 语言中，遵循的就近原则，也就是说同一块作用域内存在同名的局部变量和全局变量，那么使用的时候，程序就会就近寻找变量。
