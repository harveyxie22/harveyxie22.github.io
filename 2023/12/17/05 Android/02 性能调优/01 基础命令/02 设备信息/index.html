设备信息

Harvey Xie 2023-12-17 11:22:14

设备信息

信息 命令 备注
系统版本 adb shell getprop ro.build.version.release
系统API版本 adb shell getprop ro.build.version.sdk
设备型号 adb -d shell getprop ro.product.model
厂商名称 adb -d shell getprop ro.product.brand
序列号 adb get-serialno或者adb shell getprop ro.serialno
IMEI adb shell dumpsys iphonesubinfo 手机系统受限，不一定可获取
MAC地址 adb shell cat /sys/class/net/wlan0/address
内存信息 adb shell cat /proc/meminfo
存储信息 adb shell df
内部存储信息 adb shell df /data
SD卡存储信息 adb shell df /sdcard
获取手机分辨率 adb shell wm size
获取手机物理密度 adb shell wm density

核数

  1. 查看CPU的核数

    cat /sys/devices/system/cpu/present

  2. 查看CPU开核数

    cat /sys/devices/system/cpu/online

  3. 开核命令（依次打开编号为1、2和3的CPU）

    echo 1 > /sys/devices/system/cpu/cpu1/online
echo 1 > /sys/devices/system/cpu/cpu2/online
echo 1 > /sys/devices/system/cpu/cpu3/online

  4. 关核命令（依次关闭编号为1、2和3的CPU）

    echo 0 > /sys/devices/system/cpu/cpu1/online
echo 0 > /sys/devices/system/cpu/cpu2/online
echo 0 > /sys/devices/system/cpu/cpu3/online

频率

  1. 查看编号为０的CPU正在运行工作频率（单位：千赫兹）

    cat /sys/devices/system/cpu/cpu0/cpufreq/cpuinfo_cur_freq

  2. 查看编号为0的CPU最高工作频率（单位：千赫兹）

    cat /sys/devices/system/cpu/cpu0/cpufreq/cpuinfo_max_freq

  3. 查看编号为0的CPU最低工作频率（单位：千赫兹）

    cat /sys/devices/system/cpu/cpu0/cpufreq/cpuinfo_min_freq

  4. 查看编号为0的CPU所支持全部主频率列表（单位：千赫兹）

    cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_available_frequencies

  5. 查看编号为0的CPU在两个不同频率之间切换时所需要的时间（单位：纳秒）

    cat /sys/devices/system/cpu/cpu0/cpufreq/cpuinfo_transition_latency

  6. 查看编号为0的CPU的厂家

    cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_driver

  7. 设置编号为0的CPU频率到该CPU支持的最高频率

    //1. 先查看编号为0的CPU支持的频率
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_available_frequencies
768000 884000 1000000 1100000 1200000

//2. 把支持频率的最高频率设置给编号为0的CPU
echo 416000  > /sys/devices/system/cpu/cpu0/cpufreq/scaling_min_freq
echo 416000  > /sys/devices/system/cpu/cpu0/cpufreq/scaling_max_freq

//3. 查看结果
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_cur_freq

Governor

  1. 查看编号为0的CPU支持的Governor

     cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_available_governors

  2. 查看编号为0的CPU实际运行的Governor

     cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor

  3. 设置编号为0的CPU的Governor为userspace

    echo "userspace" > /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor

DDR

  1. 查看 DDR 的当前运行频率

    cat sys/class/devfreq/scene-frequency/sprd-governor/ddrinfo_cur_freq

  2. 查看设备支持的 DDR 运行频率

    cat sys/class/devfreq/scene-frequency/sprd-governor/ddrinfo_freq_table

  3. 固定设备的 DDR 到某个频率

    echo 0 > /sys/class/devfreq/scene-frequency/sprd_governor/auto_dfs_on_off
echo xxx > /sys/class/devfreq/scene-frequency/sprd_governor/scaling_force_ddr_freq

    xxx 表示当前设备支持的DDR运行频率中的任意一个。

  4. 查看 DDR 的策略

    cat /sys/class/devfreq/scene-frequency/sprd-governor

GPU

  1. 查看 GPU 的可用频率

    cat  sys/class/devfreq/60000000.gpu/available_frequencies

  2. 查看 GPU 的当前频率

    cat /sys/class/devfreq/60000000.gpu/cur_fre

编译方式

查看应用的编译方式，可以按照如下步骤操作：

  1. 先查看应用的包名(如已知包名可以跳过此步骤)：

    adb shell pm list packages

    或者，

    adb shell pm list packages | adb shell grep youtube

    以模糊查询youtube的包名为例。

  2. 查看相关信息：

    adb shell dumpsys package com.google.android.youtube

    将滑动条滚到到信息的Dexopt state:位置，就可以看到youtube的编译方式是：

    Dexopt state:
  [com.google.android.youtube]
    path: /product/app/YouTube/YouTube.apk
      arm64: [status=verify] [reason=prebuilt]

需要修改应用的编译方式，可以使用如下的命令(案例：修改 calculator 的编译方式为 quicken )：

> adb shell cmd package compile -m quicken com.google.android.calculator

跳过开机向导

adb shell settings put secure user_setup_complete 1
adb shell settings put global device_provisioned 1
adb reboot

Dumpsys

adb shell dumpsys gfxinfo <PACKAGE_NAME>

查看窗口信息

dumpsys window windows | grep "Window #"

获取应用 APK

adb shell pm path <PACKAGE_NAME>
